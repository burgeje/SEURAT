Generate Candidate Patterns

The users can use “generate candidate patterns” action in the context menu under a decision to create alternatives based on candidate patterns selected from the pattern library. There is a “Generate Candidate Pattern Wizard” that will assist the user through the narrowing of the patterns. The user may select the patterns they fill that are most useful to the decision, and then add them as alternatives.

 

The user starts by right clicking on a decision and select “Generate Candidate Patterns”. A wizard will appear, as shown in Figure 1.

Figure 1

If the user selects “Exact matching”, only the non-functional requirements that are directly applicable to at least one ontology entry of the pattern will affect the pattern score.
If the user selects “Contribution matching”, all non-functional requirements that are either directly or indirectly applicable to at least one ontology entry of the pattern will affect the scores.

 

In the next page, shown in Figure 2, the user needs to select the types of the pattern that will be used in the matching.

 

Figure 2

The user is allowed to choose more than one pattern types. Select at least one of them to continue.

 

In the next page, as shown in Figure 3, the user needs to select the categories of the patterns that will be used in the matching.

Figure 3

Only the problem categories that are specific to one of the types selected in the previous page will be available for selection. To select a problem, select one from the “Available Problems” and click the button “ =>”. The problem category will be removed from the list of available problems and added to the list of selected problems. If you want to remove a pattern from the selected problems, simply select them in selected problems, and click “ <=”. The problem will be removed from the list of selected problems and added back to the list of available problems.

Choose at least one of the problems and click “Next”.

 

In the last page of the wizard, the user will be asked to confirm the selection. If you are satisfied with the selections, click “Finish” and you will be presented with evaluation screen. If you would like to change the selections, press the “Back” button.

 

Figure 4

You will then be presented with a evaluation screen, as shown in Figure 5.

Figure 5

The patterns are sorted with descending order of their scores. Since the screen contains only one pattern, and no NFR’s are matching with either positive or negative ontology of the pattern Broker, the pattern Broker is displayed with a score of 0. (Perhaps we need some better examples…)

Use “=>” button to select the pattern and add it to the list of selected patterns. If you want to redo the selection, use the “Empty Selected List” button to clear the selections. After you’re finished, press “Done”. The window will close and after a brief moment, the patterns are added under the decision you have selected as alternatives.
