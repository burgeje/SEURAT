Patterns

 

 

In Figure 1, we see that a pattern contains quality attributes and sub-decisions.

A quality attribute is a claim that the pattern will affect the design either positively or negatively.

For example, the pattern “Three-layer” is claimed to support functionality reuse.

 

A sub-decision is a decision that a software designer has to make before the pattern can be adopted.

 

 

 

 

 

 

 

 

It is possible to add a new pattern, quality attribute, decisions and candidate patterns under the decisions.

To add a new Pattern, you need to right click on the pattern type that the pattern you are inserting is applicable. For example, you can right click on “Architectural Patterns” to add a new architectural pattern. You will find an editor appears.

 

Figure 2

Figure 2 contains the current edit window. Please note that only the categories that are applicable to the pattern type will be selectable in the drop down menu. After you finish adding the details about the pattern, click “Save” button to save the pattern to the library. It should appear on the pattern library tree as well as the database.

 

You can also edit an existing pattern. To edit “Three-Layer”, for example, right click on “Three-Layer” pattern and click “Edit”. You will be presented with the same window as Figure 2 but with the contents of what is already in the database. You can press “Save” to save the edits to the database.

 

You can also delete a pattern, to delete the pattern, right click on the pattern and select “Delete”. Note that if you delete a pattern, the pattern library will recursively delete all the associations between ontology entries and the pattern, and delete all sub-decisions and their associations with the pattern before deleting the pattern. After the pattern is deleted, it should no longer appear in the pattern library.

 

You can also add sub-decisions and quality attributes to the pattern. You can find the relevant actions by right clicking on the pattern. If you have selected Add Ontology, you will be presented with the ontology association screen. Please refer to relevant rationale-explorer document for details. If you have selected Add Decision, you will be presented with the decision editor. Please refer to “Sub-decision” document.
