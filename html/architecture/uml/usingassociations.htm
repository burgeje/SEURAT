UML Association – Using the system

There are several actions currently supported by the system.

·      Exporting a pattern to a UML model. This can be done in the pattern library. It only exports the pattern and does not establish an association.

·      Associating a pattern-alternative to a UML model. Exports and associates a pattern to a model.

·      Verifying a pattern-alternative in the associated model. This will test whether the model still implements the pattern.

·      Disassociating a pattern-alternative from a model. This will delete the associating in SEURAT, but it does not delete the association in the model.

·      Navigate to UML model. This allows the user to navigate from an associated pattern-alternative to the UML model.

·      Navigate to SEURAT. This allows the user to navigate from the diagram to the alternative, which will be expanded and revealed to the user in the rationale explorer tree.

 

 Exporting a pattern:

Go to the pattern library. Select the pattern you wish to export. Right click on the selected pattern. Then select “Export to XMI(UML)” (or alternative select “Export to existing…” if you want to add the pattern to an existing model.) A file selection dialog will open. Choose a file that is writable. Then the system will ask you how many instances of each role you wish to add to the model. Enter a positive integer on each dialog. The export will be finished shortly after all inputs are made.

 

Associating an alternative:

To associate a pattern, you must create a pattern-alternative from “Generate Candidate Patterns” wizard in the rationale explorer. You must adopt the pattern-alternative (make sure the adoption is saved to the database), and then right click the alternative and select “Associate UML”. The steps after that are similar to export.

After the association is completed, the system will run the verification once.

Note that once the alternative is associated with an UML, you cannot change the status from adopted from anything else. Also, you may not delete the UML. In order to do any of this, you must disassociate the alternative first.

 

Verifying an association:

After the alternative is associated with a model, it will be verified every time the alternative is updated in the SEURAT database. You can also verify the alternative manually by right clicking on the alternative and select “Verify UML”. The verification will test whether the model still implements the pattern the alternative represents. After the verification, the system will change the alternative’s icon to one of the followings:

Limitations of the verification:

The verification algorithm will violate the definition of “model implements pattern” defined in the definition page in one particular instance: If the user has added other pattern participants outside the SEURAT, the system will not add the participant to any roles. This is due to the limitations of the generative approach of handling UMLs.

 

Navigations:

To navigate from the SEURAT alternative to the model, right click on the alternative and click “Go to UML Model”.

If you want to navigate from papyrus to SEURAT, right click on the class or association in the diagram and click “To SEURAT”. (Note if you don’t see the action on a class, try right clicking on somewhere else on the class. The “properties…” selections should also show up along with “To SEURAT” selection.)
