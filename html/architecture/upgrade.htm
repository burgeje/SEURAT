Database Upgrade Instructions

New database tables are necessary for the pattern library to function. If you do not have those tables, you will not be able to open pattern library.

 

Step 1: Create Tables

MySQL Users:

Create table patterns (
Id integer not null AUTO_INCREMENT,
name varchar(255) default null,
type varchar(255) default null,
description blob default null,
problem blob default null,
context blob default null,
solution blob default null,
implementation blob default null,
example blob default null,
url varchar(255) default null,
primary key (id));

Create table patterndecisions (
Id integer not null AUTO_INCREMENT,
name varchar(255) default null,
description varchar(255) default null,
type varchar(255) default null,
status varchar(255) default NULL,

phase varchar(255) default NULL,

subdecreq varchar(255) default NULL,

ptype varchar(255) default NULL,

parent INTEGER  default NULL,

subsys INTEGER  default NULL,

designer INTEGER  default NULL,

parentpattern int,

PRIMARY KEY  (id));

 

Create table pattern_decision(
patternID INTEGER default NULL,

decisionID INTEGER default NULL,

parentType varchar(255) default NULL);

 

Create table patternproblemcategories (
id integer not null AUTO_INCREMENT,
problemcategory varchar(50) default null,
patterntype varchar(20) default null,

PRIMARY KEY  (id));

 

Create table pattern_problemcategory (
id integer not null AUTO_INCREMENT,
patternID integer default null,

problemcategoryID integer default null,

PRIMARY KEY  (id));

 

Create table pattern_ontentries (
patternID integer default null,
ontID integer default null,
direction varchar(50) default null);

Step 2: Import new data

You need the default XML file in order to import the new data. If you have the source code, the XML file is located at the relative root of the development directory ( the parent of src ), and it is named ratDBCreate.xml

You will need to go to pattern library first. Make sure that pattern library was opened successfully, and if not, exit eclipse and check whether the tables has been created. If pattern library opens, you can consult ÒImport from XMLÓ section of the manual to import rabDBCreate.xml
