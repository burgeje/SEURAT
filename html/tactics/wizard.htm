Generating Tactics

To use the tactic library in your rationale database, you will need to generate a tactic. Right click on a decision or a sub-decision does this, then click “Generate Candidate Tactics”. This will bring up the “Generate Candidate Tactics Wizard” (henceforth “Wizard”).


In the first page of the wizard, the user need to select which one of the positive quality attributes the tactic would exhibit. The user is not restricted to leaves of the argument ontology. (For example, “Dependability Criteria” was selected on the example.) If the user has selected “Argument-Ontology” (the root), for example, the system will take all available tactics in the library for consideration.

 

If you wish to evaluate against the patterns, you must have the parent of the sub-decision be an alternative that was generated from the “Generate Candidate Patterns Wizard”. Otherwise, the choice is pre-selected by the wizard and is disabled to the users. If you wish to add pattern impact scores, you may select “Yes” on the left-hand side.  In most circumstances, recursive evaluation is not desired. However, if the tactic being employed will impact everything in the system (such as modifiability tactics), then it may be useful to evaluate recursively up to the root of decision.

 

The wizard after the second page is self-explanatory.
